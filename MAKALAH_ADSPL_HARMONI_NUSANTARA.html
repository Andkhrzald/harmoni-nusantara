BAB I: PENDAHULUAN

1.1 Latar Belakang

Indonesia merupakan negara dengan keragaman agama yang tinggi. Berdasarkan data Kementerian Agama RI, terdapat 6 agama resmi yang diakui: Islam, Kristen Protestan, Katolik, Hindu, Buddha, dan Konghucu. Minimnya platform digital yang memfasilitasi pemahaman lintas iman secara moderat menyebabkan potensi miskomunikasi dan intoleransi.

Harmoni Nusantara hadir sebagai solusi platform web yang menyediakan: Edukasi keagamaan lintas iman untuk 6 agama resmi, informasi ibadah praktis berbasis API real-time, ruang diskusi moderat dengan AI chatbot, aksi sosial kolaboratif lintas agama, konsultasi privat terenkripsi, dan cek fakta berita keagamaan.

1.2 Rumusan Masalah

  1. Bagaimana merancang sistem platform moderasi dan literasi beragama yang dapat diakses oleh seluruh kalangan masyarakat?
  2. Bagaimana merancang basis data yang mampu menampung data multi-agama, multi-konten, donasi, konsultasi, dan forum diskusi?
  3. Bagaimana mengintegrasikan layanan eksternal (API Aladhan, Google Maps, YouTube, Calendarific, Fact Check, Gemini AI) ke dalam sistem?
  4. Bagaimana menerapkan keamanan data (enkripsi pesan konsultasi) pada platform berbasis web?

1.3 Batasan Masalah

  1. Sistem berbasis web (desktop & mobile responsive).
  2. 6 agama resmi Indonesia (Islam, Kristen, Katolik, Hindu, Buddha, Konghucu).
  3. Framework: Laravel 13, PHP 8.3, PostgreSQL (Supabase).
  4. Tidak membahas implementasi payment gateway secara detail.
  5. Tidak membahas deployment dan infrastruktur production secara mendalam.

1.4 Tujuan Penelitian

  1. Menghasilkan analisis dan desain sistem platform moderasi beragama.
  2. Menghasilkan rancangan ERD, LRS, dan spesifikasi file database.
  3. Menghasilkan use case diagram, activity diagram, sequence diagram, class diagram, dan deployment diagram.
  4. Menghasilkan rancangan user interface dan dokumen input/output.
  5. Menghasilkan spesifikasi pengujian perangkat lunak dan spesifikasi hardware/software.

BAB II: TINJAUAN PUSTAKA

2.1 Konsep Dasar Analisa dan Desain Sistem

Menurut Satzinger, Jackson, dan Burd (2024), analisis dan desain sistem adalah proses sistematis untuk mengidentifikasi masalah, merencanakan solusi, dan merancang sistem informasi. Metode SDLC yang digunakan adalah model Waterfall dengan tahapan: analisis kebutuhan, desain sistem, implementasi, pengujian, deployment, dan maintenance.

2.2 Unified Modeling Language (UML)

UML adalah bahasa pemodelan standar untuk menggambarkan artefak sistem perangkat lunak (Fowler, 2018). Diagram yang digunakan: Use Case Diagram (interaksi aktor-sistem), Activity Diagram (alur kerja), Sequence Diagram (interaksi objek), Class Diagram (struktur statis), dan Deployment Diagram (konfigurasi hardware/software).

2.3 Framework Laravel

Laravel adalah framework PHP berbasis MVC (Model-View-Controller) yang menyediakan Eloquent ORM, Migration & Seeder, Blade templating, Middleware, dan Livewire untuk komponen reaktif server-side.

BAB III: ANALISIS SISTEM

3.1 Analisis Sistem Berjalan

Saat ini belum ada platform terintegrasi yang menyediakan edukasi 6 agama, informasi ibadah, ruang diskusi, dan aksi sosial dalam satu aplikasi web. Masyarakat harus mengakses berbagai sumber terpisah untuk mendapatkan informasi lintas iman.

3.2 Analisis Sistem Usulan

Harmoni Nusantara mengusulkan platform terpadu dengan 7 modul: Autentikasi, Edukasi & Literasi, Asisten Ibadah, Aksi Sosial (donasi, relawan, cek fakta, konsultasi), Forum Diskusi (Ruang Bersama + AI chatbot), Testimoni Pengguna, dan Dashboard/Admin Panel.

4.4 Pengkodean (Coding Convention)

4.4.1 Primary Key

Semua tabel menggunakan BIGINT AUTO_INCREMENT via sintaks Laravel: $table->id().

4.4.2 Foreign Key

$table->foreignId('religion_id')->constrained('religion_categories')->onDelete('cascade');
Format: foreignId('nama_tabel_tunggal_id')->constrained('nama_tabel')->onDelete('aksi');
Aksi: cascadeOnDelete() atau nullOnDelete()

4.4.3 Unique Index

Diterapkan pada: email (users), slug (religion_categories & education_contents), composite (forum_room_id, user_id) di forum_participants.

BAB VII: PENUTUP

7.1 Kesimpulan

  1. Harmoni Nusantara berhasil dirancang sebagai platform moderasi dan literasi beragama berbasis web dengan arsitektur MVC menggunakan Laravel 13.
  2. Basis data terdiri dari 16 tabel yang saling berelasi mencakup modul edukasi, ibadah, donasi, relawan, cek fakta, konsultasi, forum, testimoni, favorit pengguna, dan sorotan religi.
  3. Sistem terintegrasi dengan 7 API eksternal: Aladhan (jadwal sholat), Google Maps (peta ibadah), YouTube (video edukasi), Calendarific (hari besar), Google Fact Check (cek fakta), Google Gemini (AI chatbot), dan Web Speech API (Text-to-Speech aksesibilitas).
  4. Keamanan data diterapkan dengan enkripsi AES-256-CBC pada pesan konsultasi menggunakan Crypt::encryptString().
  5. Terdapat 4 level akses pengguna (Guest, User, Penyuluh, Admin) dengan RoleMiddleware untuk otorisasi.

7.2 Saran

  1. Integrasi payment gateway (Midtrans/Xendit) untuk donasi online real-time.
  2. Pengembangan fitur notifikasi real-time menggunakan WebSocket/Pusher.
  3. Implementasi Progressive Web App (PWA) untuk akses mobile offline.
  4. Penambahan fitur multi-bahasa (Indonesia, Inggris, daerah).
  5. Deployment menggunakan Laravel Cloud untuk skalabilitas.

Lampiran 1: RoleMiddleware - Middleware otorisasi berbasis role

class RoleMiddleware {
    public function handle(Request $request, Closure $next, string ...$roles): Response
    {
        if (!in_array(auth()->user()?->role, $roles)) {
            abort(403, 'Unauthorized');
        }
        return $next($request);
    }
}
